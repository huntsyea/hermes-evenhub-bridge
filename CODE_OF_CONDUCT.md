# Code of Conduct

## Our standard

This project should be a practical, respectful place to report issues, propose changes,
and review code. Be direct about technical concerns, assume good faith, and keep feedback
focused on the work.

Unacceptable behavior includes harassment, threats, discriminatory language, personal
attacks, publishing private information, or sustained off-topic disruption.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; block users; or take other
reasonable moderation actions when behavior makes the project unsafe or unproductive.

## Reporting

Report conduct concerns privately to the maintainer at the email listed in
`pyproject.toml`. Security vulnerabilities should be reported through GitHub Security
Advisories as described in `SECURITY.md`.
